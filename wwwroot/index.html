Relatude.DB with TurboQuad from Google Research in C#

This experiment contains 30 000 wikipedia articles indexed in a vector database with 3072 dimensions from OpenAI (text-embedding-3-large). The sample contains two database instances, one using a basic flat vector index, the other the new TurboQuant algorithm.

Relatude.DB is an opensource C# native database under development. Click here for the code of this example. Click here for the TurboQuant implementation.

The TurboQuant algorithm reduce the memory by a factor of around 3x while maintaining most of the speed. (On my local machine turboquad is faster, on this Azure hosting, it is slower than the flat index, not sure why...) The search is a "brute force" implementation, and all 30 000 vectors are compared with the query vector, there is no partitioning or clustering of the vectors. It is multithreaded and the query cache is off.
